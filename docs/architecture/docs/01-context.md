# Контекст системы

## Назначение

BionicPRO — платформа для управления бионическими протезами, которая обеспечивает:

- Сбор телеметрии с протезов в реальном времени
- Формирование отчётов для пользователей
- Управление заказами через CRM
- Безопасную аутентификацию с MFA

## Пользователи системы

| Роль | Описание |
|------|----------|
| Пользователь протеза | Владелец бионического протеза, просматривает отчёты |
| Покупатель | Оформляет заказы на протезы |
| Оператор CRM | Управляет заказами и клиентами |
| ML-инженер | Анализирует данные телеметрии |
| Администратор | Управляет системой и пользователями |

## Внешние системы

- **Яндекс ID** — внешний Identity Provider для аутентификации
- **Google** — альтернативный Identity Provider
- **Бионический протез** — IoT устройство, передающее телеметрию

## Архитектурные принципы

1. **Разделение OLTP/OLAP** — CRM работает с PostgreSQL, отчёты из ClickHouse
2. **BFF Pattern** — токены хранятся на сервере, фронтенд работает с cookies
3. **CDC для real-time данных** — изменения в CRM автоматически реплицируются
4. **Многоуровневое кэширование** — Redis + S3 + Nginx CDN
